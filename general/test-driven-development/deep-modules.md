# 深度模块

出自《A Philosophy of Software Design》：

**深度模块** = 小接口 + 大量实现

```
┌─────────────────────┐
│   Small Interface   │  ← 方法少，参数简单
├─────────────────────┤
│                     │
│                     │
│  Deep Implementation│  ← 复杂逻辑被隐藏
│                     │
│                     │
└─────────────────────┘
```

**浅层模块** = 大接口 + 少量实现（应避免）

```
┌─────────────────────────────────┐
│       Large Interface           │  ← 方法多，参数复杂
├─────────────────────────────────┤
│  Thin Implementation            │  ← 只是透传
└─────────────────────────────────┘
```

设计接口时，问自己：

- 能否减少方法数量？
- 能否简化参数？
- 能否将更多复杂性隐藏在里面？
